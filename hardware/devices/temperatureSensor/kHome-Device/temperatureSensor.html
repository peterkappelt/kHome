Device Report: temperatureSensor.khd

Meta Information

  • Author: Peter Kappelt
  • Device ID: 0x1
  • Device Version: S1.0, H1.0
  • Device Comment:
    A general purpose temperature- and humidity sensor.
    Developed for kHome RF Protocol

Data Registers

Address Name Length (Byte) Initial Value Read Only Description
0x1 temperature 4 0 true This value contains the measured temperature in tenth degrees
You've to divide this value with ten to get the real value
Furthermore, this value is represented as an 32bit signed integer
0x2 humidity 4 0 true This register contains the humidity value in tenth %RH
You've to divide this value with ten to get the real value
This value is represented as an 32 bit unsigned integer

Config Registers

Address Name Initial Value Read Only Description
0x0 deviceAddress 0 false The kHome address, that the device listens to.
This may be changed by the user when initializing a new device
0x10 autoTransmitInterval 0 false The value of this register declares, how often the data registers 0x01 and 0x02 (temperature and humidity) get transmitted
A value of 0 means, that they will never be broadcasted, only read manually

Status Registers

Address Name Initial Value Description
0x0 globalStatus 1 Is an Information about the devices status in general.
If 0, everything is good.
Anything but 0 indicates a fault.
This register needs to be set to 0 after successfull initialisation in software.
0x1 deviceID 1 ID of the device.
May not be changed internally
